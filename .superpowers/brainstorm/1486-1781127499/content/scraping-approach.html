How should the tool extract brand tokens from a URL?

This is the core of the tool — everything else is templating. The approach determines output quality.

// fetch → parse CSS
GET pineapple.com/style.css
→ extract --color-primary
→ extract font-family
→ extract border-radius
// no browser needed

CSS Parser

Fetch the page's HTML and stylesheets, extract CSS custom properties and computed values. Fast, cheap, no AI needed.

Pros

  • Instant, no API cost
  • Works offline
  • Fully deterministic

Cons

  • Misses JS-rendered styles
  • Raw tokens, no narrative
  • Can't match pineapple example quality
// screenshot → Claude Vision
fetch
(url) → screenshot
claude
.analyze(screenshot,
"extract brand tokens,
colors, fonts, spacing"
)
→ design-analysis.yaml

AI Vision (Claude) — Recommended

Take a screenshot of the page, send it to Claude with the pineapple example as the quality template. Gets computed styles, understands visual hierarchy, writes the narrative. Natural screenshot fallback if URL fails.

Pros

  • Matches pineapple example quality
  • Handles JS-rendered sites
  • Screenshot fallback is the same flow
  • Writes the narrative too

Cons

  • Uses Claude API credits
  • Needs Playwright on Vercel
  • Slightly slower (~10–20s)
// CSS first, AI enrichment
tokens = parseCss(url)
analysis = claude.enrich(
tokens, screenshot
)
→ grounded + narrated

Hybrid — CSS + AI enrichment

Parse CSS for raw token values first (fast, grounded), then pass those + a screenshot to Claude for semantic labeling and narrative writing. Most accurate and most complex.

Pros

  • Grounded in actual CSS values
  • AI fills gaps and writes narrative

Cons

  • Most complex to build
  • Still needs Playwright
  • Marginal quality gain for the effort

My recommendation: AI Vision — the pineapple example is clearly not just CSS extraction; it's interpreted analysis. Claude Vision is the only way to get that quality automatically, and the screenshot fallback becomes free.
